Адаптивний дизайн

  • Респонсивна (responsive) веб-сторінка має кілька варіантів відображення, переходи між ними плавні, елементи і контейнер тягнуться, як гумові. При зміні розміру в'юпорта блоки плавно стискаються або розтягуються, а коли настає точка перелому, змінюють своє розташування так, щоб оптимально зайняти весь вільний простір по горизонталі.
  • Адаптивна (adaptive) веб-сторінка має кілька варіантів відображення. На відміну від респонсивної сторінки — це буквально різні стани. Дизайн змінюється ривками в жорстко заданих точках перелому і не тягнеться між ними.
  • Область перегляду (в'юпорт, viewport) — це видима, прямокутна область веб-сторінки, що не включає адресний рядок, панель закладок та інші службові елементи браузера.
  • Мобільні браузери відображають сайт у віртуальному в'юпорті (layout viewport). Ширина віртуального в’юпорта залежить від виробника пристрою.
  • Фізичний в’юпорт (visual viewport) — це величина, що дорівнює ширині фізичного екрану пристрою. У мобільних пристроях розмір фізичного в’юпорта за замовчуванням не збігається з віртуальним. Віртуальний в’юпорт ширший за фізичний екран пристрою.

Налаштування області перегляду на мобільних пристроях

<meta name="viewport" content="width=device-width, initial-scale=1.0" />

Медіазапити

Медіазапит - оголошується директивою @media у файлі CSS, та описує набір інструкцій, які мусять бути виконані при зміні ширини вьюпорту

@media media-type and (media-feature) {
}

Медіатип

  • screen - визначає стилі, що будуть застосовані тільки для пристроїв з фізичним екраном;
  • print - визначає стилі, що будуть застосовані при відправці на друк;
  • all - визначає стилі для усіх пристроїв (за замовчуванням)

Медіатип

  • min-width - визначає мінімальну ширину в'юпорта, відстежує зміни від зазначеної ширини вьюпорту (включно);
  • max-width - визначає максимальну ширину в'юпорта, відстежує зміни до зазначеної ширини вьюпорту (включно).
  • orientation - визначає стилі згідно з орієнтцією пристрою (значення lanscape - фльбомне, portait - портретне)
@media screen and (min-width: 900px) {
}

Логічні оператори

  • and (І) — передбачає виконання всіх умов для того, щоб твердження було істинним

    @media screen and (min-width: 400px) and (max-width: 800px) {
    }

  • or (АБО) — передбачає хоча б одну умову з набору, щоб твердження було істинним (записується через кому)

    @media screen and (max-width: 600px), (min-width: 900px) {
    }

  • not (НЕ) — передбачає інверсію, тобто твердження буде істинним від зворотного, коли умова НЕ виконана

    @media not print{
    }

Адаптивна графіка

Ретинізація графіки

Ретинізація — це техніка підготовки растрової графіки. Для цього потрібно експортувати з макета більше зображення.

  • 1. Експорту зображень у N-разів більше від розміру оригіналу
  • 2. Збереження великих зображень з відповідними префіксами @2x і @3x. Для оригіналу префікс не потрібний.
<img
srcset="./img/photo.jpg 1x, ./img/photo@2x.jpg 2x"
src="./img/photo.jpg"
alt="Опис зображення для всіх версій"
/>

srcset - список версій зображення в різних розмірах

src - вказується версія зображення в стандартній якості. Ця версія зображення буде використана у випадку, якщо браузер старіший і не вміє працювати з атрибутом srcset.

Респонсивні (гумові) зображення

img {
display: block;
max-width: 100%;
}

Фонові зображення

Медіафункції resolution, min-resolution і max-resolution визначають щільність пікселів екрана CSS

Значення медіафункцій перевірки щільності пікселів — це одиниця dpi (dots per inch), тобто кількість фізичних пікселів на дюйм екрана.

На екранах стандартної щільності пікселів в одному дюймі 96 точок. Тому для перевірки екранів із щільністю 2 і більше вказуємо значення 192dpi (96x2).

Загальні стилі

.box {
width: 200px;
height: 300px;
background-image: url('photo.png');
background-size: 200px 300px;
}

або

.box {
background-image: url('photo.png');
background-size: cover;
}

якщо щільність екрана як мінімум 2x

@media (min-resolution: 192dpi) {
.box {
background-image: url('photo@2x.png');
}
}
