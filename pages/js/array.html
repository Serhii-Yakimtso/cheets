Масиви

Масив — це впорядкована структура даних, яка використовується для зберігання колекції елементів. Можуть містити елементи різних типів даних, таких як числа, рядки, булеві значення тощо. Для створення використовуються квадратні дужки [ ]. Усередині дужок перераховуються елементи масиву, розділені комами.

const mixed = ['apple', 10, true];

Кожен елемент масиву має свій порядковий номер - індекс. Індексація елементів масиву починається з 0.
Щоб отримати доступ до елемента масиву, використовується синтаксис квадратних дужок.

const planets = ['Earth', 'Mars', 'Venus'];
console.log(planets[0]); // 'Earth'
console.log(planets[2]); // 'Venus'

Перевизначення значення елемента

const planets = ['Earth', 'Mars', 'Venus'];
planets[0] = 'Jupiter';
planets[2] = 'Neptune';
console.log(planets); // ['Jupiter', 'Mars', 'Neptune']

Довжина масиву

const planets = ['Earth', 'Mars', 'Venus'];
console.log(planets.length); // 3

Індекс останнього елемента

const planets = ["Earth", "Mars", "Venus"];
const lastIndex = planets.length - 1;
console.log(planets[lastIndex]); // "Venus"

Приведення типів

const array = [1, true, "Poly"];
console.log(String(array));// "1,true,Poly"
console.log(array + "5"); // "1,true,Poly5"
console.log(Number([])); // 0
console.log(Number([1])); // 1
console.log(Number([1, 2, 3])); // NaN

console.log(Boolean([])); // true />
console.log(Boolean([1, 2, 3])); // true

Багатомірні масиви

const array = [1, true, "Poly"];
const matrix = [
[1, 2, 3]
[4, 5, 6]
[7, 8, 9]
];
console.log(matrix[0][0]); // 1
console.log(matrix[1][2]); // 6
console.log(matrix[2][2]); // 9

Синтаксис ...spread

Синтаксис ...spread перетворює масив на список аргументів. Це доцільно, коли очікується список чисел, а не масив.
Цікаво! Синтаксис ...spread перетворює числа, які записані у форматі рядків на числа, при цьому текстові рядки залишає не змінними

const temps = [14, -4, "25", 8, 11];
console.log(temps); // [14, -4, "25", 8, 11]
console.log(...temps); // 14 -4 25 8 11
console.log(Math.max(temps)); // NaN
console.log(Math.max(...temps)); // 25

Створення об'єкту за допомогою ...spread

Операція ...spread дозволяє створити копію масиву або «склеїти» довільну кількість масивів в один новий замість методів slice() і concat()

const temps = [14, -4, 25, 8, 11];
const copyOfTemps = [...temps];
console.log(copyOfTemps); // [14, -4, 25, 8, 11]
const lastWeekTemps = [14, 25, 11];
const currentWeekTemps = [23, 17, 18];
const allTemps = [...lastWeekTemps, ...currentWeekTemps];
console.log(allTemps); // [14, 25, 11, 23, 17, 18]

Деструктуризація масивів

Синтаксис

Деструктуризоване присвоювання можна використовувати для масивів.

Після ключового слова const або let ставимо квадратні дужки [ ] Змінним будуть послідовно присвоюватися значення елементів масиву в нумерованому порядку: від 0 і до кінця масиву.

const color = [200, 255, 100];
const [ red, green, blue ] = color;

console.log(`rgb(${red}, ${green}, ${blue})`);
// “rgb(200, 255, 100)"

Значення за замовчуванням

Якщо змінних оголошено більше, ніж елементів масиву, їм буде присвоєно undefined. Щоб запобігти цьому, можна вказувати значення за замовчуванням - після імені змінної ставимо "=" і значення за замовчуванням.

const color = [200, 100, 255];
const [ red, green, blue, alfa = 0.3 ] = color;

console.log(rgba(${red}, ${green}, ${blue}, ${alfa}));
// “rgba(200, 255, 100, 0.3)"

Часткова деструктуризація

Якщо з масиву необхідно деструктуризувати тільки перші N елементів, а інші зберегти в одну змінну у вигляді масиву використовують операцію ...rest.

При цьому оригінальний масив не змінюється, у змінній otherColor буде новий масив із копіями зібраних значень.

const color = [200, 255, 100];
const [ red, ...otherColors ] = color;

console.log(red); // 200
console.log(otherColors); // [255, 100]

Пропуск значень

Елементи масиву — це набір індексованих значень. Якщо необхідно деструктуризувати тільки, наприклад, третій елемент, перші два необхідно пропустити.

const rgb = [200, 100, 255];
const [, , blue] = rgb;

console.log(`Blue: ${blue}`); // "Blue: 255"

Деструктуризація параметрів функції

Під час передачі масиву у функцію, можна деструктуризувати його елементи.

function printFruits([firstFruit, secondFruit, thirdFruit]) {
console.log(firstFruit, secondFruit, thirdFruit);
}

printFruits(["apple", "banana", "orange"]); // "apple banana orange"

Переваги деструктуризації

  • Зручне вилучення значень з об'єктів і масивів: Замість того, щоб щоразу звертатися до полів об'єкта або елементів масиву за їхніми індексами або іменами, можна одразу витягти потрібні значення у змінні.
  • Короткий і читабельний код: Деструктуризація робить код коротшим і зрозумілішим. Замість довгих виразів доступу до полів об'єктів або елементів масиву, можна відразу присвоїти значення змінним зі зрозумілими іменами.
  • Параметри функцій: При передачі об'єктів у функції, можна деструктуризувати об'єкти, щоб отримати доступ до потрібних даних. Це дозволяє явно вказати, які поля об'єкта використовуються у функції.
  • Робота з функціями, що повертають об'єкти: Якщо функція повертає об'єкт, можна відразу деструктуризувати цей об'єкт, щоб витягти з нього значення.
