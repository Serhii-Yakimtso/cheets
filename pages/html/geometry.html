Геометрия елементів

Загальне

  • content (область вмісту) — це контент елемента, наприклад текст, зображення, відео чи вкладені елементи. За замовчуванням висота (height) області вмісту визначається контентом, ширина (width) — типом елемента (блоковий або рядковий).
  • padding (внутрішній відступ) — це відстань між контентом елемента і його рамкою (border). Використовується для декоративних ефектів. Не можна вказувати негативні значення.
  • border (рамка) — межа елемента. За замовчуванням її ширина дорівнює нулю. Якщо колір рамки не встановлено, вона приймає колір основного вмісту, наприклад тексту.
  • margin (зовнішній відступ) — додає відступ ззовні елемента, від зовнішньої межі (рамки) до сусідніх елементів, тим самим розділяючи їх на сторінці. Маржину можна вказувати негативне значення. Фон елемента на зовнішній відступ не поширюється.

display тип елементу block

Блоковий елемент

  • займає всю доступну ширину рядка всередині батьківського елемента;
  • висота визначається вмістом, якщо явно не задана у властивості height;
  • починається з нового рядка, тому кілька сусідніх блокових елементів розташовані вертикально один під одним;
  • можна задавати будь-які властивості геометрії елемента: ширину, висоту, падінги, рамки та маржини.
inline

Рядковий елемент

  • ширина та висота залежать тільки від вмісту, явно задати їх не можна, тобто властивості width і height не мають жодного ефекту;
  • розташовується в рядку, доки в ньому є достатньо місця, після чого нові елементи переносяться на наступний рядок;
  • можна задавати лише горизонтальну геометрію: ліві та праві margin, padding і border, тобто вони ігнорують значення верхніх і нижніх margin, padding і border.
inline-block

Рядково-блоковий елемент (використовуються, коли рядковим елементам потрібно додати декоративні ефекти)

  • ширина і висота залежать від вмісту, але можна явно задати властивості width і height.
  • розташовується в рядку доти, поки в ньому є достатньо місця, після чого нові переносяться на наступний рядок.
  • можна задавати будь-які властивості геометрії елемента: ширину, висоту, поля, рамки та відступи.
У кнопок, посилань, зображень, спанів і інших рядкових елементів за умовчанням буде проміжок 1/4 розміру шрифту найближчого батьківського елемента. можно це виправити за допомогою display: block.
none Повністю приховати елемент, звільнивши його простір для інших, елемент вилучається з потоку документа, візуально приховується й відсутній у розкладці сторінки.

Flexbox

  • main axis — головна вісь flex-контейнера, вздовж якої розміщуються елементи

  • cross axis - поперечна вісь, яка завжди перпендикулярна головній осі

  • main-start і main-end — елементи в контейнері завжди розташовуються від main-start (початок головної осі) і до main-end (кінець головної осі).

  • cross-start і cross-end - початок і кінець поперечної осі cross axis вздовж якої розташовані рядки елементів

Grid

Grid — система компонування в CSS, що адає можливість створення двовимірної сітки для організації елементів у рядки та стовпці, дозволяючи легко контролювати розміри, позиціонування та вирівнювання елементів.

  • Grid-контейнер — елемент, якому присвоєно властивість display: grid;. Він стає контейнером для всіх елементів усередині нього, названих grid-елементами.
  • Grid-елементи — вкладені елементи всередині grid-контейнера. Вони автоматично розміщуються на сітці, відповідно до налаштувань контейнера.
  • Grid-лінії — горизонтальні та вертикальні лінії, що формують структуру сітки.
  • Grid-треки — простори між двома сусідніми grid-лініями, які утворюють рядки та стовпці.
  • Grid-області — прямокутні області, що охоплюють кілька grid-комірок. Можуть бути визначені для більш контрольованого розміщення елементів.

Специфічні значення розмірів властивостей Grid

  • fr (фракція) — показує, скільки частин доступного простору має займати кожен стовпець або рядок, порівняно з іншими елементами в сітці.

    grid-template-columns: 1fr 2fr;
    grid-template-columns: 200px 1fr 2fr;

    Використовується для адаптивних макетів чи контейнера , де потрібно рівномірно чи пропорційно розподіляти доступний простір.

  • auto — дозволяє колонці або рядку автоматично підлаштовуватися під контекст.

    grid-template-columns: auto 1fr;

    Використовується якщо необхідно, щоб ширина або висота стовпця чи рядка підлаштовувалася під розмір вмісту або коли потрібно створити комбінацію з елементами різного розміру, де деякі з них повинні залишатися пропорційними, а інші — зафіксовані під вміст.

  • функція repeat() — налаштовує повторювальний шаблон колонок і рядків та їх розмірів

    repeat(count, size)
    • count — кількість повторів. Це число або значення auto-fill чи auto-fit.
    • size — розмір кожного повторюваного стовпця або рядка. Може бути заданий у будь-якій одиниці: px, %, fr, auto, тощо.
    grid-template-columns: repeat(4, 100px);
    grid-template-columns: repeat(4, 100px 200px);

    grid-template-columns: repeat(4, 1fr);
    grid-template-columns: repeat(4, 1fr 2fr);

    grid-template-columns: repeat(4, 15%);

    grid-template-columns: 200px repeat(2, 1fr) auto ;

    Використовується коли потрібно створити однакові стовпці або рядки, щоб не дублювати код або для адаптивних сіток з різними розмірами, де використання auto-fill або auto-fit спрощує додавання елементів.

    Спеціальні ключові слова в CSS Grid, які використовуються з функцією repeat()

    • auto-fill — заповнює контейнер максимальною кількістю колонок або рядків певного розміру, навіть якщо залишаються порожні комірки. Коли ширина контейнера збільшується, додаються нові колонки або рядки відповідно до вказаного розміру.
    • auto-fit (подібний до auto-fill) — якщо залишаються незаповнені стовпці або рядки, вони "колапсують", тобто стискаються, щоб заповнити доступний простір. Це означає, що комірки розширюються, щоб зайняти більше простору, коли в контейнері залишаються порожні місця.

    Цікаво!

    • auto-fill: додає колонки чи рядки постійно, навіть якщо залишаються порожні місця.
    • auto-fit: розширює комірки, щоб заповнити порожній простір, адаптуючи їх під наявні елементи.

    Важливо! Використовується

    • auto-fill — коли потрібно заповнити сітку максимальною кількістю колонок/рядків, навіть якщо деякі з них залишаться порожніми.
    • auto-fit — коли бажано, щоб колонки/рядки автоматично розтягувалися, заповнюючи весь доступний простір контейнера.

  • функція minmax() — дозволяє встановлювати мінімальні та максимальні розміри для стовпців або рядків, що робить сітку більш адаптивною.

    minmax(min, max)

    • min — мінімальне значення, яке може приймати стовпець або рядок (значення, яке елемент не зможе зменшити навіть за браку місця);

      grid-template-columns: minmax(200px, 1fr);
      grid-template-columns: minmax(150px, auto);
      grid-template-columns: repeat(3, minmax(100px, 300px));
      grid-template-columns: repeat(auto-fill, minmax(100px, 1fr));

    • max — максимальне значення, якого може досягти елемент (якщо доступний простір збільшується, елемент може рости, але не перевищить цього значення).

    Використовується коли потрібно задати адаптивний макет, що змінює розмір у межах заданого діапазону чи зміни розміру вікна або якщо потрібно уникнути занадто маленьких або занадто великих стовпців чи рядків, встановлюючи обмеження розмірів grid-елементів.

Створення сітки grid-template-areas

grid-template-areas дозволяє створювати іменовані області (areas) в сітці, які можна легко використовувати для розміщення елементів

  • Кожна рядкова частина у властивості відповідає одному ряду сітки.
  • Імена (наприклад, header, sidebar) відповідають "областям", які потім можна призначити елементам.
  • Пробіли розділяють стовпці в рядках.
  • Символ "." використовується для позначення пустих клітинок.

Кроки налаштування

  1. Визначення структури сітки - використати grid-template-areas для створення загальної схеми макета.
  2. Призначення областей елементам - використати властивість grid-area для елементів, щоб вони відповідали визначеним областям.

Особливості

  • Пусті клітинки: можна залишити область порожньою (".");
  • Розширення областей: одну область можна розтягнути на кілька клітинок;
  • Адаптивність: разом із медіа-запитами можна змінювати схему макет.

Переваги

  • Зрозумілий код: Візуально демонструє структуру макета.
  • Гнучкість: Легко змінювати розташування елементів.
  • Адаптивність: Підходить для створення макетів, які змінюються залежно від розміру екрану.

Недоліки

  • Не підходить для складних макетів з великою кількістю унікальних елементів, які не можуть бути розбиті на блоки.
.container {
    display: grid;
    grid-template-areas:
        "header header header header"
        "sidebar . main main"
        "footer footer footer footer";
     grid-template-rows: 60px 1fr 40px;
    grid-template-columns: 200px 1fr 1fr;
}

.header {
    grid-area: header;
}
.sidebar {
    grid-area: sidebar;
}
.main {
    grid-area: content;
}
.footer {
    grid-area: footer;
}


@media (max-width: 600px) {
    .container {
        grid-template-areas:
            "sidebar header header"
            "sidebar main main"
            "sidebar footer footer";
     }
}

Додаткова інформація о grid-area

grid-area додатково використовується для визначення координат комплексно замість grid-column-start, grid-column-end, grid-column, grid-row-start, grid-row-end та grid-row.

grid-area: row-start / column-start / row-end / column-end;
  • row-start — номер лінії, з якої починається елемент по рядках (включно).
  • column-start — номер лінії, з якої починається елемент по стовпцях (включно).
  • row-end — номер лінії, де закінчується елемент по рядках (не включно).
  • column-end — номер лінії, де закінчується елемент по стовпцях (не включно).
.container {
    display: grid;
    grid-template-rows: 50px 100px auto;
    grid-template-columns: 150px 1fr 2fr;
}
