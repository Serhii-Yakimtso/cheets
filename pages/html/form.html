Форми

Форми - елменти HTML для внесеня тексту та відправки інформації до серверу або фільтрів

Елементи форми
Тег Опис тегу Атрибут Значення Опис атрибуту
<form>
</form> 		контейнер для групи пов'язаних елементів форми (немає обов'язкових атрибутів) name ім'я ім'я форми, що є унікальним на поточній веб-сторінці. Може містити символи англійського алфавіту в будь-якому регістрі, цифри, підкреслення та тире. Не можна використовувати пробіл. Використовується як на сервері, так і на клієнті при роботі з формою.
autocomplete on визначає, чи може браузер автоматично заповнювати значення всіх елементів форми.
off
novalidate - Говорить браузеру не перевіряти валідність введених даних під час відправки форми. Якщо атрибут не вказано, виконується вбудована у браузер валідація
action адреса збір даних та відправка їх на сервер, вказується шлях до файлу (не використовується, крім старих проектів)
<input> елемент для створення полів вводу type text однорядкове текстове поле, що приймає будь-які символи, не успадковує сімейство шрифтів (за замовчуванням)
email відображає користувачеві смартфона спеціальну клавіатуру для введення електронної пошти з легко доступним символом @. Браузер також може надати спосіб вибрати адресу пошти з адресної книги.
password значення пароля, який вводить користувач, буде візуально замінено на маркери з міркувань безпеки

Використовуються додатково атрибути minLength, maxLength

tel відображає віртуальну клавіатуру для введення телефонних номерів у мобільних браузерах, що містить цифри, але не літери
number створює поле, в яке можно внести лише числа, ігноруючи усі інщі символи

Використовуються додатково атрибути min, max, step, value

date поле для вибору дати
time поле для вибору часу
datetime-local поле для вибору дати і часу
range створює числовий слайдер (повзунок),

Використовуються додатково атрибути min, max, step

radio

перетворює поле вводу на перемикач (радіокнопку, вибір - один з багатьох), які завжди йдуть групами:

  • кожній радіокнопці в групі задається однакове значення атрибута name;
  • у радіокнопки не можна вводити дані - кожній з них необхідно вказати значення в атрибуті value;
  • атрибут checked вказує, який перемикач буде обраний (позначений) за замовчуванням. У групі радіокнопок в стані checked може бути тільки один елемент. За замовчуванням - перший;
  • видимий користувачеві опис кнопки здійсеюється через батьківський елемент <label>

Використовуються додатково атрибути value, checked

checkbox перетворює поле вводу на перемикач (чекбокси, вибір - багато з багатьох)
  • кожному чекбоксу в групі задається однакове значення атрибута name;
  • у чекбокси не можна вводити дані - кожному з них необхідно вказати значення в атрибуті value;
  • атрибут checked вказує, який перемикач буде обраний (позначений) за замовчуванням. У групі чекбоксів в стані checked може бути скільки завгодно елементів. За замовчуванням - немає жодного;
  • видимий користувачеві опис кнопки здійсеюється через батьківський елемент <label>

Використовуються додатково атрибути value, checked

name ім'я ім'я поля вводу запису, має бути унікальним усередині форми (обов'зковий)
id ім'я вказується в тому випадку, якщо елемент не вкладенний структурно у <label>
placeholder підказка відображає текст-підказку про дані (формат), які необхідно ввести в поле ( відображається, коли елемент <input> пустий та зникає при введенні тексту від користувача), доповнює <label>
autofocus - поле вводу автоматично отримує фокус при завантаженні (не може бути на декількох елементах сторінки одночасно)
required - позначає поле форми як обов'язкове для заповнення (щоб зробити групу радіокнопок або чекбоксів обов'язковою, необхідно задати атрибут required кожному елементу групи)
disabled - дозволяє зробити інтерактивний елемент неактивним ( елементи не отримують ефекту фокусу, а також ігнорують кліки миші)
pattern запис дозволяє написати регулярний вираз, що обмеже варіанти символів, які можно вводити (за звичай не використовується, так як виконується через JS)
pattern="[a-zA-Zа-яА-ЯїЇєЄгГ']{4}"
запис {4} вказує на кількість допустимих символів, якщо замість {4} вказати + можна ввести будь-яку кількість.
title формат встановлює підказку-формат для вспливаючого вікна-підказки при валідації (використовуєтся для <input>, в якому є атрибут pattern)
pattern="[0-9]{3}-[0-9]{3}-[0-9]{2}-[0-9]{2}"
title="XXX-XXX-XX-XX"
value значення для типів radio та checkbox: значення, яке буде передано на сервер, коли користувач відправить форму
checked - для типів radio та checkbox: вказує, який перемикач буде обраний (позначений) за замовчуванням
minLength значення для типу password: обмежує кількість символів, які можна ввести в поле
maxLength значення для типу password: обмежує кількість символів, які можна ввести в поле
min число для типів number та range: забороняє вносити числа мешні вказанного значення
max число для типів number та range: забороняє вносити числа більщі вказанного значення
step число для типу number: вказує крок зміни числа, в тому числі і не цілі. (за замовчуванням значення кроку дорівнює 1)
value число для типу number: вказує початкове значення, яке бачить користувач у полі введення
<button>
</button> 		кнопка type submit кнопка відправлення форми
reset кнопка очищення елементів форми (не використовується)
<label>
</label>		 опис (мітка) інтерактивного елементу форми (input) for ім'я Вказується значення індентифікатора елемента (<input>), до якого має відношення (за умови, якщо <input> не вкладений структурно у <label>, в іншому випадку - не потрібен))
<fieldset>
</fieldset> 		контейнер для групування декількох пов'язаних елементів у формі
<legend>
</legend> 		заголовок групи елементів у контейнері <fieldset>
<textarea>
</textarea> 		створює багаторядкове поле для вводу великої кількості тексту cols значення встановлює кількість колонок (ширину). На практиці не вказується, ширина елемента контролюється через CSS.
rows значення встановлює кількість рядків (висоту)
resize both встановлює повзунок для зміни ширини та висоти елемента
horizontal встановлює повзунок для зміни ширини елемента
vertical дозволяє встановлює повзунок для зміни висоти елемента
none розміри елемента не змінюються
<select>
</select> 		випадаюче меню, огортає теги <option> name ім'я
multiple - змінює формат випадаючого списку, дозволяє обрати декілька варіантів (користувач може це зробити із зажатою кнопкою shift або ctrl)
<option>
</option> 		елемент випадаючого меню <select>, між відкриваючим та закриваючим тегами вкладається видиме користувачеві значення value значення використтовується при відправленні форми
selected - встановлює автоматично обраний елемент (за замовчуванням вибрано перший елемент option зі списку)
<datalist>
</datalist> 		огортає набір опцій (<option>), доступних для вибору. Обране значення буде встановлено для элемента <input>, с атрибутом list id ім'я

Псевдокласи
Псевдоклас Опис Псевдокласу Приклад
:placeholder-shown налаштування властивостей поля вводу під час відображення тексту-підказки input {
    border: 1px solid orange;
}

input:placeholder-shown {
    border-color: blue;
}
:focus-within

Застосовується до елемента, щойно він сам або елементи всередині нього отримують фокус. Це дозволяє застосувати стилі на:

  • мітку
  • форму
  • окреме поле форми
 form:focus-within {
    border-color: blue;
}
:checked

дозволяє додати стилі тільки до зазначених полів (інтерактивів), застосовується до:

  • радіокнопок;
  • чекбоксів;
  • опцій випадаючого меню в селектах
 input:checked {
    color: green;
}
